# دليل إدارة المشروع على GitHub 🚀

هذا الدليل يشرح لكِ بالتفصيل كيفية رفع المشروع لأول مرة وكيفية رفع أي تعديلات مستقبلية باحترافية وأمان.

---

## 1️⃣ الرفع لأول مرة (Initial Setup)
إذا كنتِ تبدئين من الصفر لربط المجلد المحلي بـ GitHub:

1. **تهيئة المستودع المحلي:**
   ```bash
   git init
   ```
2. **إضافة الملفات:**
   ```bash
   git add .
   ```
3. **تسجيل التغييرات:**
   ```bash
   git commit -m "Initial commit: Pharmaceutical RAG System"
   ```
4. **ربط المستودع بـ GitHub (تأكدي من تغيير الرابط لرابط مستودعك):**
   ```bash
   git remote add origin https://github.com/RanaKamalDaneh/Pharmaceutical-Drug-Information-Assistant-using-Retrieval-Augmented-Generation-RAG-.git
   ```
5. **تسمية الفرع الرئيسي ورفع الكود (مع القوة لتجاوز أي ملفات افتراضية):**
   ```bash
   git branch -M main
   git push -u origin main --force
   ```

---

## 2️⃣ رفع التعديلات المستقبلية (Pushing Updates)
كلما قمتِ بتغيير كود، أو تعديل ملف README، أو إضافة بيانات جديدة، اتبعي هذه الخطوات الثلاث:

1. **تجهيز التعديلات:**
   (الأمر `.` يعني إضافة كل الملفات التي تغيرت)
   ```bash
   git add .
   ```
2. **وصف التعديل:**
   (اكتبي وصفاً مختصراً لما قمتِ بتغييره بين علامتي التنصيص)
   ```bash
   git commit -m "اكتبي هنا ما الذي قمتِ بتعديله"
   ```
3. **الرفع إلى السحابة:**
   ```bash
   git push origin main
   ```

---

## 3️⃣ أوامر مفيدة للتصحيح (Troubleshooting)

- **لمعرفة حالة الملفات وما الذي لم يُرفع بعد:**
  ```bash
  git status
  ```
- **إذا نسيتِ رابط المستودع المرتبط حالياً:**
  ```bash
  git remote -v
  ```
- **لإلغاء آخر Commit قمتِ به (قبل الرفع):**
  ```bash
  git reset --soft HEAD~1
  ```

---

## ⚠️ ملاحظات أمنية هامة (Security Tips)
- **ملف `.gitignore`:** لا تقومي بحذفه أبداً. هو المسؤول عن منع رفع ملف `.env` الذي يحتوي على مفاتيح OpenAI الخاصة بكِ.
- **مجلد `chroma_db`:** لا يُنصح برفعه لأنه كبير الحجم ويتغير باستمرار؛ الأفضل أن يقوم كل مستخدم ببنائه محلياً عبر أمر `python app.py --rebuild`.
- **البيئة الافتراضية `.venv`:** لا تُرفع أبداً؛ لأنها تحتوي على ملفات ضخمة خاصة بنظام تشغيلكِ فقط.
